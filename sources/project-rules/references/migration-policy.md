# Migration Policy

Use this source for first-party policy that governs migration from mixed upstream inputs into a clean Codex distribution.

Rules:

- do not raw-import Claude-specific agent formats
- rewrite host-specific behavior into Codex-native terms
- keep one canonical implementation per role
- demote long, risky, or host-bound material into references
- keep the final product predictable by default and more powerful through explicit opt-in layers
