# Project Rules Source

This file is the first-party source for Codex-native base policy.

Core principles:

- inspect the real repository before editing
- make the smallest correct diff
- preserve unrelated user work
- verify with real tests or say what could not be verified
- avoid guessing about tools, environments, or integration state
- keep optional integrations optional
- require explicit approval for destructive or externally visible actions

This source should stay short and portable. Long-form migration notes belong in sibling references.
