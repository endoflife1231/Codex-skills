# Codex Powerpack — пакет документации

Этот пакет документации предназначен для сборки единого дистрибутива для Codex.

На текущем этапе сам assembled результат уже находится в `dist/` и `release/` на уровне корня репозитория. Этот каталог сохраняется как documentation-and-prompt source pack.

В него закладывалось объединение следующих слоёв:

- адаптированную под Codex логику агентной работы;
- субагентов;
- пользовательский пакет Skills;
- Caveman;
- Graphify;
- профили установки;
- установщик, обновление, удаление и диагностику;
- поддержку GitHub Codespaces;
- контроль лицензий, версий и конфликтов.

## Важно

Пакет не переносит интеллект Claude в Codex. Он переносит только внешний рабочий слой:

- инструкции;
- порядок действий;
- роли;
- проверки;
- сценарии;
- инструменты.

Пользователь самостоятельно предоставляет исходные репозитории и материалы.

## Порядок использования

1. Прочитайте `FULL_DOCUMENTATION.md`.
2. Подготовьте источники по `docs/INPUT_CHECKLIST.md`.
3. Положите их в каталог `sources/`.
4. Передайте Codex файл `prompts/MASTER_PROMPT_FOR_CODEX.md`.
5. После сборки проведите ревью через `prompts/REVIEW_PROMPT.md`.
6. Выполните финальную проверку через `prompts/FINAL_VERIFICATION_PROMPT.md`.

Для текущего репозитория эти шаги уже отработаны на assembled-слое; каталог остаётся полезным как source-of-truth для подхода, требований и проверочных сценариев.

## Главные файлы

- `FULL_DOCUMENTATION.md` — полная документация проекта.
- `docs/TARGET_ARCHITECTURE.md` — целевая структура.
- `docs/INSTALLER_SPEC.md` — требования к установщику.
- `docs/TESTING_AND_ACCEPTANCE.md` — тестирование и критерии готовности.
- `prompts/MASTER_PROMPT_FOR_CODEX.md` — готовая инструкция для Codex.
